## Q2 Lazy Allocation是什么？
延续刚才提到的策略，最后终于找到L3index，但是这时我们发现它的 Valid Bit 为 0，这就是 Lazy Allocation（延迟分配）发生的时刻：🎆BOOM!!!

***什么是 Lazy Allocation？***

当你调用 malloc 申请内存时，内核为了节省资源，只在你的虚拟地址空间里划地盘，却不给你物理内存。所以我们看到了Valid Bit=0。

具体流程如下：

查表：CPU 拿着你的虚拟地址，一路查到 L3 页表的对应行（描述符）。

发现骗局：描述符里的 Valid Bit 是 0。这意味着这块虚拟地址虽然属于你，但后台还没给它分配真实的 RAM。

缺页异常 (Page Fault)：硬件立刻罢工，把控制权交给操作系统内核OS。

分配内存：内核查看自己的记录，确认这是合法请求，于是从RAM里抠出 4KB 的空间，把这块空间的物理基地址填进刚才那个 L3 描述符，并把 Valid Bit 改成 1。

重试指令：内核让 CPU 重新执行刚才那条指令。这次 MMU 再查 L3 index，发现 Valid 是 1 了，翻译成功，拿到数据！
